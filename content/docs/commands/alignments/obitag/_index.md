## Description

{{< obi obitag >}} assigns a taxonomic identity to each input sequence by searching a reference database and computing the **Lowest Common Ancestor (LCA)** of the best-matching reference sequences. It is the core taxonomic classification step in metabarcoding pipelines — typically run after demultiplexing, paired-end merging, dereplication, and denoising.

For each query sequence, {{< obi obitag >}} pre-screens the reference database using 4-mer overlap counts to identify candidate references, then refines the selection using **Longest Common Subsequence (LCS)** scoring. The best-matching references are used to compute the LCA in the taxonomy, yielding the most specific taxonomic node consistent with all top matches.

Each output sequence carries the original attributes plus the following {{< obi obitag >}}-specific annotations:

| Attribute | Description |
|---|---|
| `taxid` | Assigned taxonomic node, written as `taxon:ID [name]@rank` (e.g. `taxon:9858 [Capreolus capreolus]@species`). With `--raw-taxid`, written as a plain integer string. |
| `obitag_bestmatch` | Accession of the best-matching reference sequence. |
| `obitag_rank` | Taxonomic rank of the assigned node (e.g., `species`, `genus`, `infraorder`). |
| `obitag_bestid` | Sequence identity (ratio 0–1) of the best-matching reference. |
| `obitag_match_count` | Number of reference sequences used for the LCA computation. |
| `obitag_similarity_method` | Similarity method used: `"lcs"` for the default alignment-based mode. |

When no confident match is found (best identity ≤ 50%), the sequence is assigned to the root of the taxonomy (`taxid=1`).

## Examples

### Basic taxonomic assignment

Assign taxonomy to the ASVs from the wolf diet tutorial against the EMBL vertebrate 12S reference database:

```bash
obitag -t ncbitaxo.tgz \
       -R db_v05_r117.fasta.gz \
       wolf_query.fasta \
       --out out_basic.fasta
```

{{< code "out_basic.fasta" "text" "download" >}}

### Save and reuse the indexed reference database

Building the internal reference index takes time on large databases. Use `--save-db` to persist it once and reload it on subsequent runs:

```bash
# First run: assign and save the indexed reference DB
obitag -t ncbitaxo.tgz \
       -R db_v05_r117.fasta.gz \
       --save-db wolf_ref_indexed.fasta \
       wolf_query.fasta \
       --out out_basic.fasta

# Subsequent runs: use the pre-built index (significantly faster)
obitag -t ncbitaxo.tgz \
       -R wolf_ref_indexed.fasta \
       wolf_query.fasta \
       --out out_basic.fasta
```

### Write plain taxids with automatic deprecated-taxid correction

Use `--raw-taxid` to write compact integer taxids, and `--update-taxid` to silently replace any deprecated taxids found in the reference database:

```bash
obitag -t ncbitaxo.tgz \
       -R db_v05_r117.fasta.gz \
       --update-taxid \
       --raw-taxid \
       wolf_query.fasta \
       --out out_raw_taxid.fasta
```

{{< code "out_raw_taxid.fasta" "text" "download" >}}

### Strict taxonomy enforcement

Combine `--update-taxid` and `--fail-on-taxonomy` to first update deprecated taxids, then terminate immediately on any taxid that remains invalid after the update:

```bash
obitag -t ncbitaxo.tgz \
       -R db_v05_r117.fasta.gz \
       --update-taxid \
       --fail-on-taxonomy \
       wolf_query.fasta \
       --out out_strict_update.fasta
```

{{< code "out_strict_update.fasta" "text" "download" >}}

> **Note:** Using `--fail-on-taxonomy` alone (without `--update-taxid`) will cause {{< obi obitag >}} to exit with a fatal error when it encounters the deprecated taxids that are common in reference databases built from older taxonomy snapshots.
